Input data

 

Triangle CSV file

 

Triangle data can be imported to the reserving tool. A SQL database will store the data. The SQL database is called "triangle_data.sqlite" and is saved in the same place where the overall R components of the application are saved on your computer.

 

The triangle data should have the following structure:

*       UWYDate: This is the date of the accident or underwriting year. As a standard, 1st January can be chosen for full accident year (e.g. 1/1/2024 for accident year 2024).

*       DevDate: This the date of the development year. As a standard, 31st December can be chosen for full development year (e.g. 31/12/2024 for development year 2024).

*       AggClass: This can be used as the quarter on which the data is seen. As an example, "Q4 2024" would relate to the fourth quarter of 2024.

*       LoB: This is the Line of Business within a company. For example, "SCORCasualtyNP" would relate to the Non-Proportional Casualty line of business of SCOR.

*       DataType: DataType can be either "Incurred Claims" or "Paid Claims"

*       Value: This is the cumulative payment or incurred at development date for the selected underwriting or accident year.

 

On clicking on the button "Browse.", you can choose the csv file which you want to import. Examples of CSV file for import are also provided with the application.

 

 

Vector CSV file

 

Vector data can also be imported to the reserving tool and they will also be saved in the same SQL database.

 

The vector data should have the following structure:

*       UWYDate: Same as for triangle data

*       AggClass: Same as for triangle data

*       LoB: Same as for triangle data

*       DataType: DataType can be either "Premium" or "NbClaims" for the number of claims.

*       Value: This is the premium or number of claims for the selected underwriting or accident year.

 

On clicking on the button "Browse.", you can choose the csv file which you want to import. Examples of CSV file for import are also provided with the application.

 

 

Importing data

 

Once you have selected the CSV files you want to import, you can click on the button "Import Data". If you import only triangle data, you will see the data being imported. After the import, you should restart the application.

If you import both triangle data and vector data, it is important to make sure that UWYDate, AggClass and LoB cover the same intervals in both CSV files. Once the import is done, you should restart the application.

 

 

Retrieving loss data

 

After importing data in the reserving tool, it is possible to retrieve and see the imported data on pushing the button "Retrieve LoB from database". After pushing this button, two drop-down lists appear allowing you to select the LoB and the AggClass within the SQL database among the imported data. Once you select LoB and AggClass, you can push the button "Retrieve Reserving data". Four tabs are then created corresponding to Incurred claims, Paid claims, Premium and Number of claims. At this point, it is important to check and click on each tab to see your data (this also initiates the background variable corresponding to these 4 tabs. If you do not click on one of the tab, the background variables will not be initiated).

 

 

Excluding loss data points

 

In the tab "Incurred" triangle and "Paid" triangle, it is possible to exclude certain cells. Simply click on the cells you wish to exclude and click on the button "Exclude Selected Cells Inc" for incurred triangle or "Exclude Selected Cells Paid" for paid triangle. The projection methods will be adapted not to take into account the exclude cells.

 

 

Projecting loss data

 

The following projection methods are available:

*       Chain-Ladder (paid and incurred)

*       Bornhuetter-Ferguson (incurred only)

*       GLM (paid and incurred)

*       Munich Chain-Ladder

*       Cape-Cod (incurred only).

 

In the tab "Incurred" triangle or "Paid" triangle, just push the buttons corresponding to the chosen methods. A new tab will then appear corresponding to the projections. You can inspect the results in the relevant tab.

 

As for the Bornhuetter-Ferguson method, the projection is done in 2 steps:

*       A first step consists in entering the ultimate loss ratios and coefficients of variation of the ultimates in the tab "BF Input". If you do not enter any figure, the ultimate loss ratios are selected as 100% and the coefficients of variation are selected as 10%.

*       Once you have entered loss ratios and coefficients of variations, you can press the button "Calculate BF Inc" and a tab corresponding to the BF method will appear (tab "BF Inc").

 

The following statistics are available for each projection method:

*       Chain-Ladder:

o   Ultimate loss amounts

o   IBNRCum: IBNRs based on the ultimate loss amounts

o   Mack standard deviation and coefficient of variation: It is based on the Chain-Ladder R package and the paper of T. Mack (1993)

o   CDRCum: This is the Claim Development Result (one-year view) based on the Merz-Wuethrich method.

o   Skewness: This is based on the paper available under https://ssrn.com/abstract=2344297

*       Bornhuetter-Ferguson:

o   Bf_Ult: Ultimate loss amounts

o   Bf_Res: IBNRs based on the ultimate loss amounts

o   msepRi: Mack standard deviation according to the the paper of T. Mack (2008)

o   Skew: This is the skewness based on the paper available under https://variancejournal.org/article/29442-the-skewness-of-bornhuetter-ferguson

*       Munich Chain-Ladder:

o   The Paid and Incurred ultimates, the Paid and Incurred IBNRs as well as the ultimate P/I ratios are all based on the paper https://www.casact.org/sites/default/files/2021-07/Munich-Chain-Ladder-Quarg-Mack.pdf

*       Cape-Cod:

o   CC_ult: Ultimate loss amounts

o   CC_Res: IBNRs based on the ultimate loss amounts

o   CC_MSEPi: Standard deviation based on the paper of Anina Saluz (2015) available in https://doc.rero.ch/record/300085/files/S1748499514000359.pdf

o   Skew: This is the skewness based on the paper available under https://variancejournal.org/article/37897-the-skewness-of-cape-cod-in-a-distribution-free-model

*       GLM:

o   Ultimate loss amounts and IBNRs are provided

o   For each modality of each variable the test of goodness-of-fit is also provided to see if the GLM model has a good fit to the underlying data.

 

Once the different projection methods are run, it is possible to make a final selection of ultimates. For this purpose, press the button "Final Selection" in the tab "Incurred". A new tab will appear called "Final Selection". In this tab, for each accident or underwriting year, it will be possible to select your preferred projection method. Two tables are available to help with the selections:

*       A table which provides the overall Ultimate loss amounts and the sum over all accident or underwriting years,

*       A table with the Coefficients of Variation per accident or underwriting year and the total Coefficient of Variation. On selecting different methods, the overall Coefficient of Variation is automatically recalculated based on a correlation matrix of the accident/underwriting years between each other. It may differ from the overall Coefficient of Variation that is calculated for the different methods (Chain-Ladder, BF, Cape-Cod, GLM),

 

 

Version management

 

Once you have selected the methods for each accident/underwriting year, you can save the resulting version of the reserving projections on clicking on the button "save results". This only saves the triangles including the exclusions done in the Incurred and Paid tabs, the Coefficients of Variations and Ultimate loss ratios selected for the Bornhuetter-Ferguson method (incurred) and the final selected projection methods.

 

You can recover a version on clicking the button "Retrieve Results". A tab called "Version retrieval" is then created and, when selecting the desired version, the triangles, Bornhuetter-Ferguson and Selections are then retrieved and shown in different tables.

 

Finally, you can also delete a version on clicking on the button "Delete Version".

 

 

Create Excel Sheets

 

Also, after selecting the projection methods for each accident/underwriting year, it is possible to export all the created tabs in excel. After pushing the button "Create Excel Sheets", an excel workbook called "tables.xlsx" is created in the folder where the R application is saved. All tabs are saved.

 

 

Starting a new projection module

 

It is always possible to restart the application on clicking the button "Delete all tabs". All tabs (except the Help tab) will be deleted.
